# ADR-000 Template

- Status: Proposed
- Date: YYYY-MM-DD
- Owners: <team or individual>

## Context
Describe the technical and organizational context.
List the constraints, assumptions, and current architecture baseline.
Reference related PRD, incidents, or prior ADRs when relevant.

## Decision
State the selected option clearly and concretely.
Define boundaries: where this decision applies and where it does not.
Include security and operational constraints.

## Options Considered
1. Option A: summary, benefits, drawbacks.
2. Option B: summary, benefits, drawbacks.
3. Option C: summary, benefits, drawbacks.

## Consequences
- Positive outcomes expected from this decision.
- Tradeoffs accepted by the team.
- Known risks and follow-up actions.

## Validation
- How the decision will be tested in implementation.
- Metrics or signals used to confirm success.
- Review date for revisiting this ADR.
